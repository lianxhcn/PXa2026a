Skill 是否能够被 AI 自动识别和调用，关键在于 Skill 是否位于 AI 工具的默认扫描目录，或者是否已经完成对应的路径配置。在 Windows 环境下，Codex/Claude Code 的默认目录如下：`C:\Users\用户名\.codex\` 或 `C:\Users\用户名\.claude\`。如果将 Skill 安装在 D 盘的自定义目录，AI 无法通过默认扫描发现该位置，因此需要手动将 Skill 路径添加进去，否则无法自动发现和调用。

### Q4. 目前还是不知道 MCP 是什么，她和 tool 的关系是什么？好像有的人就不太关注这些，有的文章又特别强调，需要专门去安装吗？

Tool（工具）是“具体能力”，比如读取文件、搜索网页、查询数据库、运行 Python、操作 GitHub；MCP 是让 AI 连接和调用这些工具的一套统一接口标准，类似于 USB-C：工具是鼠标、硬盘、显示器，MCP 是统一的插口和通信规则。MCP 服务器可以向 AI 提供工具、数据资源和提示模板，而 AI 所在的应用作为客户端去发现并调用它们。

现在不需要为了使用 Codex 专门安装 MCP。若只是写代码、运行 MATLAB/Python、编辑本地项目、生成 PDF，Codex 自带的终端、文件和 Skills 通常已经够用。只有希望 Codex 直接连接某个外部系统，例如数据库、Notion、特定科研平台或某个未内置的软件，而且对方提供了 MCP Server，才需要配置或安装它。

至于有些文章特别强调 MCP，是因为它们讨论的是 AI Agent、工具集成或自动化系统开发；有些人不关注，是因为普通聊天、写作、数据分析或本地编程并不要求用户理解 MCP。所以说，如果现在Tool就够用的话其实不用专门安装MCP。

### Q1. 数字字典一般是由谁提供？自己编辑数据库时应如何设计数字字典

大家可以先将原始数据的变量名以及已有的数据口径信息整理给AI，也可以直接利用 AI 工具（如 Claude Code、Codex 等）对数据进行初步探索。在此基础上，建立和完善数据字典：
（1）首先，让 AI 根据数据结构、变量名称和已有信息生成一版数据**字典草稿**，包括变量含义、可能的单位、取值范围、缺失值情况等；
（2）随后结合自己的研究需求和对数据的理解，对 AI 生成的结果进行**核对和修改**。例如，AI 可能无法准确判断某些变量的具体含义，这时需要进行补充说明；如果发现数据中存在缺失值、空格、异常值等问题，也需要先进行数据清洗，再让 AI 根据清洗后的数据进一步更新和完善数据字典。
（3）最终，后续的实证分析应基于清洗后的数据和最终确认的数据字典，这样可以保证变量定义清晰、数据处理过程可追溯，也方便后续研究复现。
另外，在课件"**2 数据处理、探索性分析与样本构造：2.2.4 数据字典**"部分，连老师给了详细的prompt 和推荐的 skills 大家可以安装使用。
（回答人：陈云菲）

呃，如果你的原始数据是从c码a或者是统计局拿来的，通常他们都会配了一个说明书，那个说明书呢就一两页，把所有的变量都列出来，呃，后面跟着这个变量的中文含义计算方法。你基本上只需要把这个说明书丢给AI，它就可以把初步的数据字典帮你建立起来。

如果你的数据是爬虫得来的，那你在爬虫之前就先要跟AI说清楚，说我们要爬哪些字段，每一个字段我要按照一个标准的数据字典的方式把它记下来，你会告诉他这个数据字典里边的字段呢就包括数据的衡量单位呃缺失的情况，然后从哪些网站爬下来的。这样在爬取数据的同时，你就已经有了一个初步的数据字典，方便后续的数据清洗和分析。就后续你在分析的过程中，可能还会利用你已经得到的原始数据衍生出来一些新的指标，包括对有些变量取对数，还有一些变量相除，得到一些比率，这些新生成的变量，你要及时的在代码里边加一些注释语句。这个时候你的AI呢，回头可以根据你的注释语句和你生成变量的那个计算方式帮你去补充那个数据字典。

所以一个比较完整的流程就是你一开始跟AI做计划来帮你做数据清洗的时候，就加一条规则，你说你可以在每个阶段处理完以后都帮我把这个阶段新生成的变量做成一个完整的数据字典的格式，那个数据字典格式呢，你最好能够定义一个模板，或者是你从别的人那儿看到他的数据字典做的很漂亮，你就放到一个文件夹里面，你说你要跑到那个文件夹里面去参考那个数据字典的格式，在我们整个数据清理的过程中呃，只要有新变量产生，你就帮我记录到那里面去。


### Q2. 示例数据很少，可以逐条查看合并失败的记录；真实数据量很大时，如何检查和处理合并问题？

数据量大时不需要逐条人工查看，而要把核对过程程序化。
（1）合并前用 isid、duplicates report 检查主键，批量统一键变量的大小写、空格、数据类型和前导零；
（2） 合并后，先用tab _merge查看整体匹配情况，再将_merge != 3 的未匹配记录单独保存，按键值或出现频数进行汇总，是被异常类型并进一步核查。
连老师课程 2.4 中强调的核心思路就是：先检查数据的粒度和匹配键，再检查合并后的匹配率。
AI可以辅助生成批量诊断代码、整理未匹配记录的共同特征，但合并方式、键变量的实际含义，以及未匹配样本是否应保留，仍需由研究者判断。不能让AI直接使用 m:m、模糊匹配，或自行删除未匹配数据。
（回答人：刘嘉伟）

真实数据量较大时，可以先让 AI 或代码工具统计合并失败的数量和比例，并总结**主要问题类型**。之后可以区分两类处理：
（1）适合**批量处理**的部分：例如统一变量格式、去除空格、大小写转换、处理缺失值、按照规则进行标准化匹配等，可以让 AI 协助编写代码批量完成。
（2）需要**人工检查**的部分：例如判断两个名称是否代表同一个对象、企业更名是否需要合并、模糊匹配结果是否合理等，让Ai 列出表格，然后进行人工确认。

### Q3. 对于回归中的控制变量，是否也需要考虑处理极端值？如果控制变量本身分布与XY直方图差异较大（例如X和Y偏正态分布、控制变量左右偏离或非钟形分布）是否可以纳入回归？控制变量本身如果进行缩尾处理后是否还需要取对数？

答：控制变量同样需要检查极端值。
（1）首先，结合描述统计、箱线图和影响力指标进行判断，极端值是录入错误、异常观测，还是具有真实经济含义。
（2）控制变量的分布不需要与 X、Y 分布相似，OLS也不要求控制变量必须呈正态分布。
（3）是否取对数应根据变量性质决定。例如收入、资产等非负且右偏明显的变量通常考虑取对数，而比例变量、虚拟变量、含负值变量通常不能直接取对数。
（4）缩尾和取对数解决的是不同问题：缩尾主要降低极端观测的影响，取对数主要缓解偏态、缩小量级差异，或者改变系数的经济解释，所以缩尾后仍可根据需要取对数。
（回答人：陈云菲）

### Q8. 在数据处理阶段，应该先缩尾还是先删除变量缺失值呢？如果后续研究中需要补充最新年份数据，是否需要重新合并所有年份样本，再操作缩尾和缺失值处理，还是单独对最新一年处理好后append进去之前年份即可？缩尾的时候是否建议by year？

答：（1）一般先处理明显错误值、构造变量并确定最终研究样本，再对连续变量进行缩尾。虽然缩尾命令通常会自动忽略该变量的缺失值，但如果之后还要因其他核心变量缺失而删除样本，缩尾分位点可能会发生变化，因此最好基于最终分析样本计算。

### Q12. 如果有些生态学类数据同一个处理组有多个重复样方处理，是直接把他们都当成对应y值做回归还是需要整合处理成一个x只对应一个y，以及一般如何整合处理，例如天然草原情况下同一种X处理下不同样方内调查到的草物种以及对应指标数目都不一样怎么处理。eg: 放牧量X为2，重复1内5个取样，五个取样里面植物物种不一定一样，有三组这样的重复，共15组200多条数据发病率Y对应时怎么处理
